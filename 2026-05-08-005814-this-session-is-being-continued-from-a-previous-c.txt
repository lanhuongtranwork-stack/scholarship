● I found the issues. There are two problems:

  1. run.py defaults to reload=True in production — because Railway doesn't set
  ENVIRONMENT, so is_dev evaluates True and uvicorn starts with --reload, which
  can cause the process to behave unpredictably in containers.
  2. Migrations running inside the lifespan block uvicorn from binding the port
  until they finish, which can race against Railway's health check timeout.

  The fix: move migrations to Railway's releaseCommand (runs before the app
  starts), and use uvicorn directly in startCommand.

● Pushed. Railway sẽ tự động redeploy. Đây là những gì thay đổi và tại sao fix
  này sẽ hoạt động:

  Root cause: run.py mặc định reload=True vì Railway không set ENVIRONMENT env
  var → uvicorn spawn thêm watchdog process → không ổn định trong container.
  Đồng thời migrations chạy trong lifespan chặn uvicorn bind port, race
  condition với Railway's health check.

  Fixes:

  ┌────────────────┬─────────────────────┬──────────────────────────────────┐
  │                │        Trước        │               Sau                │
  ├────────────────┼─────────────────────┼──────────────────────────────────┤
  │                │ python run.py (có   │ uvicorn main:app --host 0.0.0.0  │
  │ startCommand   │ reload=True)        │ --port $PORT (trực tiếp, không   │
  │                │                     │ reload)                          │
  ├────────────────┼─────────────────────┼──────────────────────────────────┤
  │                │ Chạy trong lifespan │ Chạy trong releaseCommand trước  │
  │ Migrations     │  → block port       │ khi container start              │
  │                │ binding             │                                  │
  ├────────────────┼─────────────────────┼──────────────────────────────────┤
  │ Seeding        │ Chạy trong lifespan │ Chạy trong releaseCommand        │
  │                │                     │ (python seed_countries.py)       │
  ├────────────────┼─────────────────────┼──────────────────────────────────┤
  │ Health check   │ 60s                 │ 300s                             │
  │ timeout        │                     │                                  │
  └────────────────┴─────────────────────┴──────────────────────────────────┘

  Khi Railway deploy: trước tiên nó sẽ chạy releaseCommand (migration + seed),
  sau đó mới start uvicorn. Uvicorn lúc này sẽ start gần như tức thì và trả lời
  /api/health ngay, không cần chờ gì cả.
